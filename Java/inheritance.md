## 상속이란?

* 상속은 객체 지향 프로그래밍의 핵심 요소 중 하나로, 기존 클래스의 필드와 메서드를 새로운 클래스에서 **재사용**하게 해준다.
* 상속을 사용하려면 extends 키워드를 사용하면 된다. 그리고 extends 대상은 하나만 선택할 수 있다.
  * 부모 클래스 (슈퍼 클래스): 상속을 통해 자신의 필드와 메서드를 다른 클래스에 제공하는 클래스
  * 자식 클래스 (서브 클래스): 부모 클래스로부터 필드와 메서드를 상속받는 클래스

<br>

## 상속이 필요한 이유

```
예를 들어, 전기차( ElectricCar )와 가솔린차( GasCar )를 만들었다고 생각해보자.
전기차는 이동( move() ), 충전( charge() ) 기능이 있고, 가솔린차는 이동( move() ), 주유( fillUp() ) 기능이 있다.
전기차와 가솔린차는 자동차( Car )의 좀 더 구체적인 개념이다. 반대로 자동차( Car )는 전기차와 가솔린차를 포함하는 추상적인 개념이다.
둘은 이동( move() )이라는 공통 기능이 존재한다. 전기차든 가솔린차든 주유하는 방식이 다른 것이지 이동하는 것은 똑같다.
이런 경우 상속 관계를 사용하는 것이 효과적이다.
```

<br>

## 상속과 메모리 구조

* **자식 클래스의 객체를 생성하면, 상속 관계에 있는 모든 부모 클래스의 객체를 포함해서 생성된다.**
  * 참조값은 하나이지만, 실제로 그 안에서는 자식 클래스를 포함한 모든 부모 클래스가 함께 포함해서 생성된다.
  * 외부에서 볼때는 하나의 인스턴스를 생성하는 것 같지만, 내부에서는 부모와 자식이 모두 생성되고 공간도 구분된다.
* 상속 관계의 객체를 호출할 때, 대상 타입을 정해야 한다. 이때 호출자의 타입을 통해 대상 타입을 찾는다.
* 현재 타입에서 기능을 찾지 못하면 상위 부모 타입으로 기능을 찾아서 실행한다. 기능을 찾지 못하면 컴파일 오류가 발생한다.

> 상속 관계의 경우, 객체 내 메서드나 필드를 호출하면 부모 클래스와 자식 클래스 중 어떤 인스턴스에서 먼저 호출될까?

이때는 **호출하는 변수의 타입(클래스)을 기준으로 선택**한다.
상속 관계에서는 **자식 타입에 해당 기능이 없으면 부모 타입으로 올라가서 찾는다.**

만약, 변수의 타입이 자식 클래스인 ElectricCar라고 가정해보자.
이 변수에서 charge() 메서드를 호출하면 인스턴스 내부에 같은 타입인 ElectricCar를 통해서 charge()를 호출한다.
또한 move() 메서드를 호출하면, 우선 인스턴스 내부에 같은 타입인 ElectricCar 클래스를 확인한다.
그러나 ElectricCar 에는 move() 메서드가 없기 때문에, 이 경우 ElectricCar 의 부모인 Car 로 올라가서 move() 를 찾는다.
부모인 Car에 move() 가 있으므로 부모에 있는 move() 메서드를 호출한다.
만약 부모에서도 해당 기능을 찾지 못하면 더 상위 부모에서 필요한 기능을 찾아본다.
이때, 계속 찾아도 없으면 컴파일 오류가 발생한다.
